Part 1 — Serial Monitor
What is the Serial Monitor?

The Serial Monitor is a text window inside the Arduino IDE. Your Arduino sends messages to it over the same USB cable you use to upload code — no extra hardware needed.

🛠️
Your #1 debugging tool. If your robot misbehaves, print sensor values and you'll instantly see what the Arduino is actually reading. Think of it like a chat window between your computer and the board.
⚠️
The baud rate in your code must match the rate selected in the Serial Monitor — or you'll see garbled characters. Safe default: 9600. Faster option: 115200.
The 3 Serial functions you need
begin
Serial.begin(9600)
Put in setup(). Starts the connection at the chosen speed. Nothing prints without this.
print
Serial.print(value)
Sends text/number — stays on the same line. Great for labels: Serial.print("Dist: ")
println
Serial.println(value)
Same as print but adds a new line — each reading appears on its own row, keeping output readable.

Part 2 — I²C Protocol
What is I²C?

I²C (Inter-Integrated Circuit) lets Arduino talk to multiple devices using just two wires: SDA (data) and SCL (clock).

Each device has a unique address like 0x27. Arduino calls that address and only the matching device responds — all others stay quiet.

✅
Connect an LCD, compass, gyroscope, and temperature sensor all to the same two wires — no extra pins per device. The bus is shared; addresses keep devices separate.

🔍 I²C Address Scanner
My LCD isn't working — wrong address?

Most LCD modules use address 0x27, but some ship with 0x3F. Upload this sketch and it will print the real address of every connected I²C device.

🔍
Upload this sketch, open Serial Monitor at 9600 baud, and it prints the address of every I²C device found. Use that exact address in your LiquidCrystal_I2C constructor.
#include <Wire.h>
void setup() {
  Wire.begin();
  Serial.begin(9600);
}
void loop() {
  for(byte a=1; a<127; a++) {
    Wire.beginTransmission(a);
    if(Wire.endTransmission()==0)
      Serial.println(a, HEX);
  }
  delay(5000);
}

⚠️ Common Mistakes to Avoid
❌
Wrong baud rate in Serial Monitor Code says 9600 but monitor is 115200 → garbled output. Always match both.
❌
Missing LiquidCrystal_I2C library Install "LiquidCrystal I2C" by Frank de Brabander via Arduino Library Manager.
❌
Wrong I²C address lcd(0x27) but module is 0x3F → blank screen. Run the scanner to find the real address.
❌
Forgetting lcd.init() or lcd.backlight() Without init() the LCD ignores commands. Without backlight() it's on but invisible.
⚠️
No delay() inside loop() Serial output floods too fast to read. Add delay(500) or delay(1000) between prints.
⚠️
Not clearing LCD between updates Old characters show through. Call lcd.clear() or pad with spaces before reprinting.
