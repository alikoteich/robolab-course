🔧 Assembly Steps

Click each step to check it off. Complete all 7 steps before uploading code.

1
Assemble the chassis
Attach all 4 DC motors to the chassis frame using screws. Fit wheels onto motor shafts. The chassis should roll freely on all 4 wheels.
2
Mount Arduino & L298N
Stick Arduino Uno and L298N motor driver onto the chassis top plate using double-sided tape or M3 screws. Keep them close to reduce wire length.
3
Connect motors to L298N
Left motors (both) → Motor A terminals. Right motors (both) → Motor B terminals. Twist motor wire pairs together to reduce interference.
4
Mount & wire HC-SR04
Mount the ultrasonic sensor at the front, facing forward. Wire: VCC→5V, GND→GND, TRIG→Pin 9, ECHO→Pin 10.
5
Wire Arduino → L298N
Pin 5→ENA, Pin 6→ENB, Pin 7→IN1, Pin 8→IN2, Pin 3→IN3, Pin 4→IN4. Double-check each wire against the table before continuing.
6
Connect 9V battery
Connect battery holder to L298N power input (+9V and GND). The L298N's 5V output can power the Arduino via the 5V pin.
7
Final check before power-on
Confirm all connections match the wiring table. No bare wires touching. Battery disconnected during wiring — connect only when ready to test.

Assemble the chassis
Attach all 4 motors and wheels. All wheels should spin freely and motors firmly fastened.
2
Mount Arduino, L298N & HC-05
Secure all three boards. Mount HC-05 where its antenna has clear line-of-sight — not buried in metal or near motor coils.
3
Wire motors to L298N
Left motor pair → Motor A on L298N. Right motor pair → Motor B on L298N.
4
Wire HC-05 to Arduino
VCC→5V, GND→GND. HC-05 TXD → Arduino RX (Pin 0). HC-05 RXD → Arduino TX (Pin 1) via voltage divider (1kΩ + 2kΩ). TX and RX always cross!
5
Wire Arduino → L298N
Pin 5→ENA, Pin 6→ENB, Pin 7→IN1, Pin 8→IN2, Pin 3→IN3, Pin 4→IN4.
6
Pair HC-05 with your phone
Power on. Go to phone Bluetooth settings → select HC-05 → enter 1234 or 0000. Open BT serial app and connect. HC-05 LED slows down when connected.
7
Final check
Disconnect RX wire from Pin 0 before uploading code. Reconnect after upload. Then connect battery and test commands from phone.

💻 Arduino Code

Copy into Arduino IDE and upload via USB. Read the comments — they explain every line. 

// ══ OBSTACLE AVOIDANCE CAR ══

// Pin Definitions
const int trigPin = 9;   // HC-SR04 trigger
const int echoPin = 10;  // HC-SR04 echo
const int ENA=5, ENB=6;  // Motor speed (PWM)
const int IN1=7, IN2=8, IN3=3, IN4=4;
long duration; int distance;

void setup() {
  pinMode(trigPin,OUTPUT); pinMode(echoPin,INPUT);
  pinMode(ENA,OUTPUT); pinMode(ENB,OUTPUT);
  pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT);
  Serial.begin(9600);
}

void loop() {
  distance = getDistance();
  Serial.println(distance);
  if (distance < 20) {   // Obstacle!
    stopMotors(); delay(300);
    turnLeft();   delay(500);
  } else {
    moveForward();           // Clear path
  }
}

int getDistance() {
  digitalWrite(trigPin,LOW); delayMicroseconds(2);
  digitalWrite(trigPin,HIGH); delayMicroseconds(10);
  digitalWrite(trigPin,LOW);
  duration = pulseIn(echoPin,HIGH);
  return duration * 0.034 / 2;
}
void moveForward() {
  analogWrite(ENA,180); analogWrite(ENB,180);
  digitalWrite(IN1,HIGH); digitalWrite(IN2,LOW);
  digitalWrite(IN3,HIGH); digitalWrite(IN4,LOW);
}
void turnLeft() {
  analogWrite(ENA,160); analogWrite(ENB,160);
  digitalWrite(IN1,LOW); digitalWrite(IN2,HIGH);
  digitalWrite(IN3,HIGH); digitalWrite(IN4,LOW);
}
void stopMotors() {
  digitalWrite(IN1,LOW); digitalWrite(IN2,LOW);
  digitalWrite(IN3,LOW); digitalWrite(IN4,LOW);
}

// ══ BLUETOOTH CONTROLLED CAR ══

const int ENA=5,ENB=6,IN1=7,IN2=8,IN3=3,IN4=4;
char cmd;

void setup() {
  Serial.begin(9600);
  pinMode(ENA,OUTPUT); pinMode(ENB,OUTPUT);
  pinMode(IN1,OUTPUT); pinMode(IN2,OUTPUT);
  pinMode(IN3,OUTPUT); pinMode(IN4,OUTPUT);
  analogWrite(ENA,200); analogWrite(ENB,200);
}

void loop() {
  if (Serial.available() > 0) {
    cmd = Serial.read();
    switch(cmd) {
      case 'F': moveForward(); break;
      case 'B': moveBack();    break;
      case 'L': turnLeft();   break;
      case 'R': turnRight();  break;
      case 'S': stopMotors(); break;
    }
  }
}
void moveForward() { digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); }
void moveBack()    { digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); }
void turnLeft()    { digitalWrite(IN1,LOW);digitalWrite(IN2,HIGH);digitalWrite(IN3,HIGH);digitalWrite(IN4,LOW); }
void turnRight()   { digitalWrite(IN1,HIGH);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,HIGH); }
void stopMotors()  { digitalWrite(IN1,LOW);digitalWrite(IN2,LOW);digitalWrite(IN3,LOW);digitalWrite(IN4,LOW); }

✅ Test & Debug

Something not working? Diagnose it with this checklist.

🚗 Obstacle Car
✅ First: Serial Monitor
Tools → Serial Monitor (9600 baud). Should show distance numbers. Seeing 0 or garbage? → Check TRIG/ECHO wiring.
⚙️ Motors not spinning?
Check: (1) Battery on L298N? (2) ENA/ENB wired? (3) Motor wires tight in terminals? (4) IN1–IN4 all connected?
🔄 Doesn't avoid obstacles?
Distance always 0 → TRIG/ECHO swapped. Always >20 → sensor not facing forward. Try threshold 30 cm instead of 20.
🔧 Motor goes wrong way?
Swap the two wires in that motor's terminal (Motor A+ ↔ Motor A−). No code change needed — this is mechanical.
📱 Bluetooth Car
✅ Can't pair HC-05?
Power robot first (HC-05 LED blinks fast). Go to phone BT settings, select HC-05, enter 1234 or 0000. Pair from settings, not from app.
⚙️ Paired but no response?
Check: (1) RX disconnected before upload? (2) App sending uppercase F/B/L/R/S? (3) Baud = 9600? HC-05 slow blink = connected ✅
🔄 Wrong direction?
Swap the two wires in that motor's terminal. If left/right swapped, swap IN1&IN2 with IN3&IN4.
📡 Signal drops?
Stay within 5–8 m. Keep HC-05 antenna exposed. Add 100nF capacitors across motor terminals to reduce noise.
